## 🖥️ What this tool does

better-seo-js helps you set up SEO data for a web app in one place. It brings together page tags, social preview data, JSON-LD, sitemap support, and image metadata. That means your site can give search engines and social sites the right details without you setting everything by hand.

It is built for modern apps, including Next.js projects. It uses a typed SEO model, so the SEO settings stay neat and easy to manage. The core has no extra dependencies, so it stays light.

## 📦 Main features

### 🏷️ Typed SEO model

The project uses a typed model for SEO settings. That helps keep your fields organized and reduces bad inputs.

### 🌐 Next.js Metadata support

It works with Next.js Metadata so you can set page data in a format that fits modern Next.js apps.

### 🧾 JSON-LD output

JSON-LD gives search engines more context about your content. This can help with rich snippets and structured data.

### 🖼️ OG image and icons workflow

The tool can help prepare Open Graph images and icons, so your pages look right when shared.

### 🧱 Zero-dep core

The core stays free of extra dependencies. That keeps the setup lean and easier to ship.

### 🗺️ Sitemap support

A sitemap helps search engines find your pages. This project includes a path for generating and managing sitemap data.

## 🛠️ How to use with a Next.js app

If your app uses Next.js, you can place the SEO output in your app’s page metadata. A common setup is:

1. Create your SEO data file.
2. Add title and description values for each page.
3. Add Open Graph fields for share cards.
4. Add JSON-LD for the page type.
5. Connect the output to your Next.js Metadata config.
6. Check the page in the browser.
7. Test the page link in social preview tools.

If you use a shared layout, you can keep default site values there and override them on each page.
